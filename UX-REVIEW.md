# StoryFit UX 점검 & 개선 로드맵

> 작성: 2026-06-01 · 대상: `apps/web` 전 화면 · **UX 관점 집중**
> 우선순위: 🔴 P0(핵심 마찰·즉시) · 🟠 P1(높음) · 🟡 P2(중간) · ⚪ P3(폴ish)
> 분류: `[내비]` `[채팅]` `[모바일]` `[접근성]` `[온보딩]` `[피드백]` `[일관성]`

이 앱은 **Win98/레트로 데스크톱 메타포**(데스크톱 + 하단 Dock + 시작 메뉴 + 창)를 채택. 데스크톱에선 440px 폭의 "폰 창"이 좌측에 떠 있고, 모바일에선 전체화면. 아래는 그 톤을 유지하면서의 개선안.

---

## 🔴 P0 — 핵심 마찰 (주력인 채팅 경험에 직접)

### P0-1. `[채팅][내비]` 대화 중 AI 모델을 바꿀 수 없음
- `handleModelChange`(`page.tsx:412`)는 존재하지만 **이를 호출하는 UI가 채팅 화면에 없음**. `AI_MODELS`는 메시지 뱃지 표시용으로만 쓰임(`:652`).
- 결과: 사용자가 대화 도중 Gemini↔다른 모델 전환 불가. 생성 페이지에서 고른 모델로 고정.
- **조치:** 채팅 헤더(⚙ 옆)에 모델 드롭다운(`AiPill`/`AI_MODELS` 활용) 추가. 이미 핸들러가 있으니 UI만 연결하면 됨.

### P0-2. `[채팅][피드백]` 스트리밍 본문이 완료 후 "다른 내용"으로 교체됨
- 스토리/롤플레이에서 `needsResponseRevision`에 걸리면 스트리밍이 끝난 뒤 재작성본으로 **메시지가 통째로 바뀜**. 사용자는 읽던 글이 갑자기 달라지는 경험.
- **조치:** 재작성이 도는 동안 "✦ 다듬는 중…" 같은 짧은 상태 표시 후 교체. 최소한 교체 시 깜빡임/페이드로 "바뀌었음"을 인지시키기.

### P0-3. `[모바일][채팅]` 사이드패널이 열리면 채팅이 찌그러짐
- `.side-panel`(`globals.css:808`)이 `chat-layout` flex 안 220px **in-flow** 컬럼. 360px 화면에서 패널을 열면 채팅 본문이 ~140px만 남음. 오버레이(`page.tsx:994`)는 닫기용이지만 패널 자체는 떠 있지 않음.
- **조치:** 모바일(≤680px)에서 `.side-panel`을 `position: fixed` 우측 슬라이드오버(전체 높이, 폭 80vw)로. 데스크톱은 현행 유지.

### P0-4. `[내비][일관성]` 뒤로가기 동선이 어긋남
- 채팅 헤더 "←"(`page.tsx:558`) → `/`(홈). 대부분 **채팅 목록**에서 들어왔는데 홈으로 튕김.
- 캐릭터 페이지 "← 뒤로" → `/`, 캐릭터 생성 "← 취소" → `router.back()` 으로 제각각.
- **조치:** 채팅 "←" → `/chatlist`(또는 `router.back()`). 뒤로가기 타깃 규칙 통일.

## 🟠 P1 — 높음

### P1-1. `[내비]` 상시 탐색 수단 부재 (Dock은 "현재 위치"만 표시)
- 하단 Dock의 탭은 **현재 페이지 한 개만** active로 보여줄 뿐, 섹션 간 이동은 매번 "시작" 메뉴를 열거나 홈을 거쳐야 함. 모바일에서 탭 수 증가.
- **조치:** Dock에 항상 보이는 핵심 탭(채팅목록 / 캐릭터 / 새 대화 / 설정) 고정 배치. 레트로 톤 유지하면서 "작업표시줄 고정 아이콘"처럼.

### P1-2. `[접근성]` 아이콘 전용 버튼에 접근성 이름이 전무 (`aria-label` 0건)
- 🎒 STAT ⚙ 🔊 🎤 ✏ ⑂ ✕ ← 등 아이콘 버튼에 `aria-label` 없음(전 `(main)` 페이지에서 0건). 스크린리더 사용자는 용도 인지 불가.
- 입력 필드의 `<label className="label">`는 시각용일 뿐 `htmlFor`/`id` 연결 없음 → 프로그램적 라벨 부재.
- **조치:** 아이콘 버튼에 `aria-label`, 입력에 `id`+`htmlFor`(또는 `aria-label`) 부여. 비용 작고 효과 분명.

### P1-3. `[피드백]` 모달 키보드/포커스 처리 없음
- `ConfirmDialog`·분기 모달·온보딩 모달이 **Escape로 안 닫히고**, 열릴 때 포커스 이동/트랩 없음. 위험한 삭제 확인에서 Enter/Esc 기본 동작 기대가 어긋남.
- **조치:** 공통 `<Modal>` 래퍼로 Escape 닫기 + 첫 버튼 autofocus + 포커스 트랩. ConfirmDialog 먼저 적용.

### P1-4. `[채팅]` 대화 설정(⚙) 패널이 한 화면에 과밀
- 분기설명 · 페르소나 · 핵심메모리 · 타임라인 · 로어북(추가/편집) · 장기메모리(승격)를 220px 한 컬럼 스크롤에 전부. 초심자가 압도됨.
- **조치:** 접이식 섹션(아코디언) 또는 상단 탭("기억 / 세계관 / 분기")으로 묶기. 자주 안 쓰는 분기설명은 접어두기.

### P1-5. `[채팅]` 메시지 액션 발견성 (탭해야 나타남)
- 액션(재생성/편집/분기/삭제)은 메시지를 탭/호버해 `activeId`로 토글해야 노출. 신규 사용자는 "메시지를 누르면 메뉴가 뜬다"를 모름.
- **조치:** 첫 사용 시 마지막 AI 메시지에 살짝 힌트(예: 재생성 버튼 상시 노출) 또는 길게누르기 힌트 토스트 1회.

## 🟡 P2 — 중간

### P2-1. `[온보딩]` 가이드 1단계 링크가 모호
- 온보딩 STEP1 "캐릭터 만들기"가 `/characters`(선택 화면)로 감 — 정작 만들기는 `/characters/new`. 프리셋이 있어 바로 시작 가능한 건 장점이나, 카피와 목적지가 약간 어긋남.
- **조치:** STEP 카피를 "캐릭터 고르기/만들기"로, 또는 목적지를 의도에 맞게.

### P2-2. `[일관성]` "페르소나" 개념의 분산
- `/personas`는 `/characters`로 리다이렉트만(`personas/page.tsx`). 내 역할(페르소나) 설정은 오직 채팅 ⚙ 패널 안에만 존재 → "내가 누구로 플레이하는지"가 잘 안 보임.
- **조치:** 채팅 헤더에 현재 페르소나 칩 표시(이미 이름은 표기) + 탭으로 빠른 변경, 또는 새 대화 생성 시 페르소나 선택을 더 부각.

### P2-3. `[피드백]` 로딩 상태가 텍스트 스피너 위주
- 목록/캐릭터/대화 로드가 "불러오는 중..." 텍스트. 스켈레톤 UI로 바꾸면 체감 속도·완성도↑.

### P2-4. `[채팅]` 입력창 편의
- 멀티라인 textarea(Enter 전송/Shift+Enter 줄바꿈)는 양호. 추가로: 전송 중 입력 임시 보관, 길이/토큰 힌트, 마지막 입력 ↑키 복원.

### P2-5. `[모바일]` 데스크톱 440px 프레임의 우측 여백
- `@media(min-width:900px)`에서 shell `max-width:440px` + 좌측 패딩 140px → 큰 화면에서 우측이 크게 빔. 의도된 레트로 연출이지만, 옵션으로 "넓게 보기"(max-width 상향) 토글 제공 검토.

## ⚪ P3 — 폴리시

- `[피드백]` 토스트가 단일 메시지 — 액션별 아이콘/색 구분(성공/실패).
- `[채팅]` TTS 음성·속도 선택 UI(현재 ko 보이스 자동 선택, 고정 rate 1.0).
- `[접근성]` 색 대비 점검(특히 `--ink-soft` 텍스트, opacity 0.55 토큰 표시).
- `[채팅]` 선택지 키보드 단축키(1~4 숫자키로 선택).
- `[일관성]` 빈 상태/에러 일러스트 톤 통일(이모지 → 픽셀 아이콘).
- `[온보딩]` 첫 스토리 대화에서 stats/inventory·로어북 기능을 살짝 코치마크로 소개.

## 추천 진행 순서

1. **P0-4 (뒤로가기) + P0-1 (모델 전환 UI)** — 작고 즉시 체감.
2. **P0-3 (모바일 사이드패널) + P1-2 (aria-label)** — 모바일·접근성 기본기.
3. **P0-2 (재작성 교체 피드백)** — 주력 모드 체감 품질.
4. **P1-1 (Dock 상시 탭) + P1-4 (패널 정리)** — 탐색·정보구조.
5. **P1-3 (모달 공통화) + P2-3 (스켈레톤)** — 완성도.
