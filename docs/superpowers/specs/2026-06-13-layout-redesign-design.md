# StoryFit 전면 레이아웃 재설계 (2026-06-13)

전 화면을 단일 다크 디자인 언어로 재정비한다. 브라우저 목업으로 화면별 합의를 마친 결과의 확정 스펙.

## 목표 / 범위
- 데스크톱 메타포·다중 테마 잔재를 걷어내고 **다크 모드 단일**의 일관된 모바일 UI로 통일.
- 홈을 발견+이어하기 통합 피드로 개편, 화면 간 카드/헤더/독 디자인을 공유.
- 기존 기능은 유실 없이 유지(특히 채팅목록·채팅방·설정 패널의 세부 기능).
- 범위 밖(이번에 손대지 않음): 설정/관리자 페이지의 레이아웃(다크 CSS만 적용).

## 디자인 언어
- 배경 `#0d0d0d`, 패널 `#15151b`, 보더 `#23232b`. 액센트 보라 `#8b5cf6`, 핑크 `#e1306c`. 모바일 폭 기준(데스크톱도 동일 프레임).
- **대화 카드(공용)**: 상단 대화방 사진 + 우상단 센터 배지, 본문에 제목 `채팅방명(캐릭터.페르소나)`, 마지막 메시지, 우하단 `모드 · 시간`. 캐릭터 다수면 `(하준,서아….지수)`처럼 `…` 처리, 카드 크기 균일.
- 센터 배지 색: WHIF `#8b5cf6`, ZETA `#7c5cff`, MELTING `#ff2e93`.

## 전역 요소
- **헤더**: `로고 + StoryFit` 텍스트(모든 페이지 공통, 클릭 시 홈). 우측 🙂 계정 아이콘.
- **🙂 계정 메뉴**(드롭다운): 🔧 관리자(admin만 노출) / ⚙️ 설정 / ⏻ 로그아웃.
- **하단 독(5탭)**: 🏠 홈 / 💬 채팅 / 🧭 탐색 / 📚 서재 / 🤖 챗봇. 채팅방에서는 몰입을 위해 독 숨김.
- 첫 방문 온보딩 모달(`sf_onboarded`) 유지.

### 홈에서 유실되던 연결의 재배치 (검증 완료)
- AI 채팅 `/assistant` → 독 **🤖 챗봇 탭**으로 복귀(기존 설정 탭 자리 대체).
- 관리자 `/admin`, 로그아웃 → **🙂 계정 메뉴**.
- 설정 → 🙂 계정 메뉴(독 탭에서 제외).

## 화면별 스펙

### 홈
스크롤 단일 피드, 위에서부터:
1. **이어하기**: 가장 최근 1건만. 좌측 사진(카드 높이에 맞춰 위아래 꽉 채움) + 제목 `채팅방명(캐릭터.페르소나)` + 우상단 센터명 + 마지막 대화 2줄 + 우하단 최근 플레이 시간.
2. **센터**: WHIF/ZETA/MELTING 그라데이션 카드(이모지 없음).
3. **액션 4개**: 📖 가이드 · ✨ 새 대화(강조) · 🎭 캐릭터 · 📥 가져오기.
4. **대화중**: 진행중 대화만(대기/완결 제외) 카드 그리드, 최대 4개 + "전체 ›"(→채팅). 우하단 `모드 · 시간`.
5. **완결**: 서재 최근 등록순 카드 그리드 + "서재 ›". 우하단 `모드 · 완결일`.

### 채팅 (목록)
- 리스트형. 헤더 진행중 개수 + ☑ 선택 + ✦ 새 대화.
- 검색(제목/캐릭터명) + 입력 시 **본문 검색 결과 패널**(대화 제목·완결 태그·날짜·`역할 · 스니펫`하이라이트, `?msg=` 점프).
- 필터: 모드(전체/스토리/멀티) + 소스(STORYFIT/ZETA/MELTING/WHIF) + 정렬(최신/가나다).
- 행: 썸네일 + 제목 `채팅방명(캐릭터.페르소나)` + 마지막 메시지 + `모드 · (장) · 시간`, 우측 센터 배지 + 핀.
- 핀 고정(좌측 핑크 보더), 설정 필요 배지(`isAutoCreated` → 클릭 시 `/conversations/new?from=`).
- 액션: 스와이프 + 인라인(📌 핀 / 📚 서재로 보내기 / 🗑 삭제), 선택 모드(체크박스·전체선택·일괄삭제·ConfirmDialog).

### 탐색 (외부 센터 전용)
- 내 캐릭터는 여기 없음 → 홈 "캐릭터" 액션(`/characters`)으로. (페르소나는 별도 엔티티 아님 — 아래 유실 방지 체크리스트 참고)
- 상단 센터(WHIF/ZETA/MELTING) + 각 센터별 "최근 등록" 한 줄 가로 스와이프(최대 6 + 더보기 ›).
- 항목 카드: 사진 + 타입 배지(세계관/플롯/캐릭터) + 이름 + 간략 설명(2줄).

### 서재 (완결작)
- 홈 완결과 동일 카드 그리드(2열). 완결 편수 + 정렬 + 모드 필터.
- 카드 좌상단 ⋯ 메뉴: ↩ 꺼내기(unarchive) / 📄 소설 내보내기 / 🗑 삭제. 클릭 → 읽기 화면 `/library/:id`.

### 채팅방 (스토리 / 멀티스토리)
- 몰입형: 자체 헤더, 하단 독 숨김.
- **헤더**: ← 뒤로 · 캐릭터 사진 · `이름 · 페르소나 ▾` · 줄2 `스토리 · 3장 · 턴 12`(plain 텍스트) · 우측 **☰ / ⚙**.
  - ☰ 메뉴: STAT 스탯 / 🎒 인벤토리 / 📜 줄거리 / 📞 음성 통화.
  - 장·턴은 버튼이 아닌 텍스트.
- 상태줄(🎬 현재 상황 ▾) 별도 박스. 분기 칩(v1·v2 + ✕).
- 본문: 지문(좌측 서술) + `이름 : "대사"` / `'속마음'`, 끝에 선택지 4개. 주사위 배지 `🎲 스탯·주사위 → 결과`.
- **유저 메시지 = E안**: 우측 정렬 + 오른쪽 보라 세로바(말풍선 없음).
- 추천답변: 가로 칩(✏ 편집) 유지. 컴포저: ＋메뉴 · 입력 · 전송. ＋메뉴(음성입력/스탯판정/관전/줄거리요약/글자크기) 기존 유지.

### 캐릭터 카드 모달 (참여자/페르소나 탭 시)
- 아바타·성별 / 태그 / 세부 설정 / 시작 메시지 / 예시 대화(펼치기).
- **수정 = 이름 옆 ✏ 연필**(프리셋이면 숨김). **우측 하단 닫기만**(상단 ✕ 제거).

### ⚙ 대화 설정 패널 (채팅방)
- 우측 슬라이드 패널. **탭 3개: 기본 / 기억 / 세계관**. (연출 탭·테마·배경 이미지 제거)
- **기본**: 대화 제목 / 대화 참여자(아코디언 접힘) / 내 역할 페르소나(아코디언 접힘) / 🎨 스타일(아코디언, 접힘 시 `3인칭·과거형·…` 요약) / 분기 설명. 참여자·페르소나·스타일 모두 접힘 기본.
- **기억**: 핵심 메모리 / 타임라인(텍스트) / 🧠 장기 메모리(선택→핵심 승격, 삭제) / 🔖 북마크.
- **세계관**: 시나리오 배경 / 🔖 AI 자동 챕터(토글) / 🗺 스토리 설계도(전개방식·설계도 보기·재설계·삭제·챕터목록) / 📖 로어북(추가·가져오기·우선순위·탐색깊이·편집).

### 챗봇 (어시스턴트, 독 탭)
- **목록**: 롤플레이 없는 AI 채팅 기록(봇 아바타·제목·마지막 메시지·시간·삭제) + ✦ 새 대화.
- **대화방**: 유저=우측 보라 말풍선, AI=좌측 봇 아바타 + 말풍선(일반 서식). 스토리 요소 없음. 헤더(← · 봇 · 제목✏ · `AI 채팅 · Gemini` · ✦ 새 대화), 컴포저(입력+전송).

### 설정 페이지 / 관리자 페이지
- **레이아웃 그대로 유지, 다크 모드 CSS만 적용.** 별도 재설계 없음.
- 설정 탭: **테마 탭 제거 → 5탭**(프로필·프롬프트 / 파라미터 / 보안 / 통계 / 내보내기).
- 진입: 🙂 계정 메뉴 → 설정 / 관리자.

## 제거되는 기능
- **다중 테마 시스템**(THEMES, 외부 테마 CSS, 설정 테마 탭, 설정 패널 테마 선택) — 다크 단일.
- **대화방 배경 이미지**(customBg / `sf_bg_` localStorage / 설정 패널 배경 입력).

## 유실 방지 체크리스트 (구현 시 필수 보존)
기존에 로직은 구현됐으나 새 뷰에서 진입점이 끊길 위험이 있는 항목. 화면 구현 시 반드시 연결할 것.

### 채팅방 (목업에 안 그려졌지만 보존 필수)
- 메시지 탭 액션 세트: **재생성 / 편집 / 분기 / 삭제 / 북마크 / 음성재생(TTS)** (`MessageList`).
- **재회 인사 배너**: `comebackElapsed` + "인사 받기 / 무시" (continue comeback 모드).
- 컴포저 부가 입력: **슬래시 커맨드(`/`) 메뉴**, **`!` 시스템 커맨드**(상태창/스탯/인벤/도움말 — AI 호출 없이 즉시 응답), `↑`키 마지막 입력 복원, 글자 수 힌트.
- 메시지 영역: **이전 메시지 더 보기(페이지네이션)**, "새 답변 ↓", 본문검색 `?msg=` 점프 하이라이트, 타이핑 아바타, 주사위 굴림 오버레이.
- **sendError 재시도** 버튼.
- 헤더 ☰ 메뉴의 STAT/인벤토리/줄거리/통화는 각각 기존 StatsPopover / InventoryPopover / RecapPopover / VoiceCallOverlay로 연결.

### 페르소나 = Character (별도 엔티티 아님)
- 이 앱에 페르소나 전용 관리 UI는 없음. 캐릭터는 `/characters`에서 생성, **페르소나 지정은 채팅방 설정 패널·대화 생성 시**에만 발생. `/personas`는 `/characters` 리다이렉트(유지).
- 따라서 탐색에서 페르소나를 뺀 것은 유실 아님(원래 전용 페이지 없음).

### 테마/배경 제거 시 동반 정리 (누락 시 스타일 파손)
- **선행 필수**: 현재 다크 토큰이 `body[data-art="dark"]` 스코프에 있음 → `:root` 기본으로 승격해야 테마 제거 후에도 스타일 유지.
- 제거 대상: `lib/theme.ts`(THEMES·applyTheme·getSavedTheme·EXTERNAL_THEMES), `public/themes/*.css`, `layout` PRE_PAINT_THEME·`data-art`, 설정 ThemeTab + 설정 테마 탭, SidePanel 테마 select·배경 입력, `/api/user/settings`의 theme 필드, chatroom `customBg`(`sf_bg_` localStorage).

### 기타 진입점 보존 확인
- 새 대화 생성 플로우(`/conversations/new`), 가져오기(홈 액션·`/characters` URL·각 센터), 가이드(`/guide`), 첫 방문 온보딩 모달.
- 자동생성 대화(`isAutoCreated`) 클릭 → `/conversations/new?from=`.
- 설정 5탭·관리자 10개 하위 페이지: 레이아웃 유지(다크 CSS만), 진입은 🙂 계정 메뉴.

## 구현 메모
- 대규모 다중 화면 작업 → 화면 단위로 구현 플랜을 쪼개는 것을 권장(예: 전역 셸/독/헤더 → 홈 → 채팅목록 → 탐색 → 서재 → 채팅방+설정패널 → 챗봇 → 테마/배경 제거 → 설정·관리자 다크 CSS).
- 제목 `채팅방명(캐릭터.페르소나)`는 `conversation.title` + 참여 캐릭터명 + `personaCharacter.name` 조합. 캐릭터 다수 시 `…` 말줄임.
- 채팅방 "하단 독 숨김"은 `(main)/layout`에서 채팅방 경로일 때 `<Dock>` 미렌더로 처리.
- 목업 파일: `.superpowers/brainstorm/<session>/content/*.html` (home-v7, chatlist-v2, explore-v3, library, chatroom-v4, char-card-modal-v2, settings-basic-v2/memory/world, settings-page, chatbot-list, chatbot-room).
